EVM using Face Recognition

2024
Repository
opencv arduino python

The EVM using Face Recognition is a dual-layer security voting system designed to address the vulnerabilities of traditional Electronic Voting Machines (EVMs), such as identity fraud and multiple voting. By bridging hardware control with computer vision, this project ensures that only authenticated individuals can access the voting interface.

The system operates as a standalone offline unit, eliminating dependencies on internet connectivity while maintaining a secure local audit trail. It employs a "verify-then-vote" architecture: the Arduino hardware remains locked until the Python-based backend successfully validates the voter's credentials and facial biometrics.

Key Features

  • Biometric Authentication: Utilizes OpenCV algorithms (Haar Cascade & LBPH) to perform real-time face detection and matching against a pre-trained dataset.
  • Dual-Factor Verification: Requires a valid unique Voter ID followed by a facial match confirmation (confidence threshold 50-70) to unlock the system.
  • Duplicate Vote Prevention: Automatically locks the Voter ID in the database immediately after a vote is cast, physically preventing repeated voting attempts.
  • Hardware-Software Integration: Features seamless serial communication between the Python backend (logic/vision) and Arduino Uno (physical control).
  • Secure Data Logging: Records timestamped vote data (Voter ID + Candidate Selection) into local immutable text files for transparency and auditing.

Architecture & Workflow

The system follows a strict linear security pipeline to ensure process integrity:

  • Credential Check: The voter enters their ID via the laptop interface. The system verifies existence in the local database.
  • Facial Scan: Upon ID validation, the webcam activates. The Python script captures the face and compares it with stored biometric data.
  • Hardware Activation: If the match is valid, a signal is sent via Serial (USB) to the Arduino Uno, which "wakes up" the physical voting buttons (connected to Digital Pins 8, 9, and 10).
  • Vote Casting: The user presses a physical button to select a candidate. The Arduino captures this input and relays it back to the computer.
  • Confirmation: The vote is logged, and the Arduino blinks a Red LED (Digital Pin 6) to provide immediate visual confirmation to the user.

Hardware & Tech Stack

  • Microcontroller: Arduino Uno (ATmega328P) handles input/output peripherals and logic states.
  • Vision Stack: Python with OpenCV library for face detection and recognition.
  • Interface: Physical push buttons for candidate selection and LED indicators for status feedback.
  • Communication: PySerial library establishes the data bridge between the laptop (processing unit) and the Arduino.

Real-World Applications

  • Secure Local Elections: Ideal for municipal or panchayat elections requiring low-cost, high-trust solutions.
  • Institutional Voting: Student union or faculty elections where identity verification is mandatory.
  • Corporate Governance: Internal board member selections or secure employee surveys.

A full-stack Content Management System (CMS) backed by distributed SQL cloud infrastructure.

IIIT-NR Blogs is a dedicated digital publishing platform designed to foster community engagement within IIIT Naya Raipur. It functions as a centralized hub where students and faculty can share technical insights, campus updates, and creative writing.

Unlike static sites, this application is a dynamic, database-driven CMS. It leverages TiDB Cloud (a distributed SQL database) to ensure high availability and scalability of content. The backend logic handles the full lifecycle of a blog post—from creation and rendering to deletion—providing a seamless interface for content management.

Key Features

  • Cloud-Native Persistence: Connects to TiDB Cloud via AWS endpoints, utilizing a distributed SQL architecture for robust data storage beyond local SQLite limitations.
  • Full CRUD Functionality: Implements complete Create, Read, Update, and Delete operations, allowing authorized users to manage posts directly through the web interface.
  • ORM Integration: Uses Flask-SQLAlchemy to abstract database interactions, treating blog posts as Python objects (Blogpost class) rather than raw SQL queries.
  • Dynamic Templating: Features a responsive frontend powered by Jinja2, allowing for server-side rendering of variable content like timestamps, authors, and subtitles.
  • Secure Connectivity: Enforces SSL/TLS verification (ssl_verify_cert=true) for all database transactions to ensure data integrity during transit.

Architecture & Tech Stack

The application follows a standard Model-View-Template (MVT) architecture common in Python web development.

  • Backend: Flask (Python) for routing, request handling, and session management.
  • Database: TiDB Cloud (MySQL-compatible) hosted on AWS Southeast Asia nodes.
  • Driver: PyMySQL for compliant database connectivity.
  • Frontend: HTML5/CSS3 injected with Jinja2 template logic.

Logic & Workflow

Model Definition: The schema is defined as a Python class (Blogpost), automatically mapping attributes like title and content to database columns.

Request Handling:

  • GET /: Queries the database, orders posts by date.desc(), and renders the homepage.
  • POST /addpost: Captures form data, creates a new Blogpost object, and commits the transaction to TiDB.
  • POST /deletepost: Identifies a record by primary key (id) and executes a session delete.

Sample Code (Database Configuration)

# Database Configuration & Model
# utilizing TiDB Cloud for distributed storage
app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql+pymysql://USER:PASS@gateway01.ap-southeast-1.prod.aws.tidbcloud.com:4000/test?ssl_ca=/etc/ssl/cert.pem&ssl_verify_cert=true'

class Blogpost(db.Model):
    __tablename__ = 'blog'
    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    title = db.Column(db.String(50), nullable=False)
    content = db.Column(db.Text, nullable=False)
    date = db.Column(db.DateTime, nullable=False, default=datetime.utcnow)

@app.route('/addpost', methods=['POST'])
def addpost():
    # ORM-based insertion
    post = Blogpost(
        title=request.form['title'],
        subtitle=request.form['subtitle'],
        author=request.form['author'],
        content=request.form['content']
    )
    db.session.add(post)
    db.session.commit()
    return redirect(url_for('index'))
